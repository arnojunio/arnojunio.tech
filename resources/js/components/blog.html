Salve salve! Arnô aqui.

Então pessoas, nesse post vou mostar uma configuração muito útil caso você esteja desenvolvendo e precisa ter seu código sincronizado tanto com o repositório de controle de versão, quanto no servidor de produção, utilizando a ferramenta gti-ftp.

Algumas IDEs pagas, já implementam este tipo de rotina automaticamente, o que facilita e muito na sincronização de arquivos e aumenta a sua produtividade.

Instalando o git-ftp

Para instalar o git-ftp no linux, basta digitar o código abaixo

$ sudo apt-get install git-ftp

E pronto!

Iniciando um controle de versão em um projeto.

git init

git add seusarquivos

git commit -m "sua mensagem de commit"

git push seubranch

Configurando o git-ftp

git config git-ftp.url ip_do_servidor_ftp
git config git-ftp.user seuusuarioftp
git config git-ftp.password suasenhaftp

Inicianto o git-ftp

git ftp init

Este comando irá enviar todos os arquivos que foram versionados (repositório) para o servidor web via ftp.

git ftp push

De agora me diante, basta digitar o comando acima sempre que fizer um push pro repositório, que os arquivos alterados serão enviados para o servidor web.
